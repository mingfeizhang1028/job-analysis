# Job Analysis Dashboard

一个面向求职场景的岗位分析与简历匹配工作台。项目基于 Streamlit 构建，围绕岗位数据清洗、JD 标签抽取、岗位画像、关系网络分析、简历匹配和求职策略建议，帮助候选人从大量岗位信息中识别更适合投递的机会。

## 项目功能

- 岗位数据导入与预处理：支持读取 Excel 岗位数据，并进行公司、岗位、标签等字段的标准化处理。
- 岗位标签与 JD 结构化：结合规则与可选 LLM 能力，从岗位描述中抽取技能、业务职责、行业场景和软素质标签。
- 岗位去重与总览分析：识别相似岗位，展示岗位数量、关键词覆盖、标签分布等核心指标。
- 岗位画像与明细查询：按岗位、公司、城市、标签等维度筛选和查看岗位详情。
- 岗位关系网络：基于标签、岗位和公司之间的共现关系生成网络视图，辅助发现岗位簇和能力组合。
- 简历匹配分析：支持粘贴或上传简历文本，生成候选人画像，并计算岗位匹配度和定位差距。
- 候选人知识库：可将候选人资料切分、向量化并检索，用于后续证据召回和求职材料优化。
- LaTeX 简历生成：提供简历内容生成与模板渲染相关模块，便于输出结构化简历材料。
- Boss 直聘岗位采集辅助：包含手动采集与捕获岗位数据的辅助模块。

## 目录结构

```text
job_dashboard/
├── app.py                  # Streamlit 应用入口
├── app_core.py             # 数据加载、处理、筛选与公共渲染逻辑
├── modules/                # 核心业务模块
├── page_views/             # 各页面视图
├── utils/                  # 页面与关键词等通用工具
├── data/                   # 规则字典、示例数据与本地运行数据
├── lib/                    # 前端静态依赖
├── module_specs/           # 模块说明文档
└── 数据爬取代码/             # 岗位采集相关脚本
```

## 快速开始

1. 创建并激活 Python 环境。

```bash
python -m venv .venv
.venv\Scripts\activate
```

2. 安装依赖。

```bash
pip install -r requirements.txt
```

3. 启动应用。

```bash
streamlit run app.py
```

默认会读取 `data/jobs.xlsx`。也可以在页面侧边栏上传自己的岗位 Excel 文件进行分析。

## LLM 配置

项目中的 JD 结构化、技能抽取、简历解析和标签增强功能可以使用本地 Ollama 或远程兼容接口。相关配置位于应用侧边栏和 `data/llm_settings.json`。

如果不启用 LLM，项目仍可使用规则抽取、岗位筛选、基础统计和网络分析等功能。

## 数据与隐私说明

仓库默认不提交运行缓存、日志、个人简历知识库原始文件、LLM 运行结果和 Python 编译缓存。上传公开仓库前，请确认 `data/jobs.xlsx` 及其他数据文件中不包含不适合公开的信息。

## 适用场景

- 批量分析招聘网站采集的岗位数据
- 对比岗位 JD 中的高频技能和职责
- 根据个人简历筛选优先投递岗位
- 为简历改写、岗位定位和求职策略提供数据依据
