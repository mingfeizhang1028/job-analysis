# app 入口说明

## 变更记录

### 2026-04-20
- 修改文件：`app.py`
- 修改内容：
  - 重构应用入口缓存策略，新增“运行与缓存控制”区。
  - 默认优先复用当前会话缓存与磁盘缓存，不再因 Streamlit 重跑自动全量重新处理。
  - 仅在用户点击“重新计算当前数据”时，才对当前数据源执行全量重算并覆盖缓存。
  - 为处理结果增加 `meta.json` 元数据记录，保存缓存版本、数据集标识、处理参数、生成时间。
  - 增强数据源识别：默认文件按文件修改时间/大小识别，上传文件按内容 hash 识别。
  - 保留并接入当前会话中的 `llm_enriched_df`，支持页面级手工增强结果优先复用。
  - 新增“删除当前数据缓存”“清除当前会话增强结果”操作，避免误用旧结果。
- 影响范围：
  - `app.py` 主入口加载与处理流程
  - `data/app_cache/` 下处理结果与元数据文件
  - 页面切换 / rerun / 代码微调后的数据复用行为
- 后续整理建议：
  - 后续可将缓存能力下沉为独立模块，如 `modules/app_cache_manager.py`，避免入口文件继续膨胀。
  - 若后续页面级增强结果需要跨会话自动复用，可继续统一纳入 app 级缓存快照体系。
